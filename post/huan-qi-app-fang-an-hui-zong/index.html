URL SCheme

Android(Deep Link)

  • DeepLink用到的核心技术就是：URL SCheme

  • 步骤一:添加 intent 过滤器

 <!--要想在别的App上能成功调起App，必须添加intent过滤器-->
            <intent-filter>
                <!--协议部分，随便设置 http://bocichina.com/bci_path-->
                <data
                    android:host="bocichina.com"
                    android:pathPrefix="/bci_path"
                    android:scheme="http"/>
                <!--下面这几行也必须得设置-->
                <category android:name="android.intent.category.DEFAULT"/>
                <action android:name="android.intent.action.VIEW"/>
                <category android:name="android.intent.category.BROWSABLE"/>

 </intent-filter>
    
    备注:
    data:指定scheme
    VIEW:以便能够从 Google 搜索中访问此 intent 过滤器
    BROWSABLE:浏览器打开
    DEFAULT:隐士响应

  • 步骤二:验证是否有效:使用 adb 测试 intent 过滤器 URI

    • 使用 adb 测试 intent 过滤器 URI
    adb shell am start
            -W -a android.intent.action.VIEW
            -d <URI> <PACKAGE>

    ego:

    adb shell am start -W -a android.intent.action.VIEW -d "http://bocichina.com/bci_path" com.example.androidapp11

  • 唤醒方式一:app内唤醒

    //跳转app
Intent action = new Intent(Intent.ACTION_VIEW);
StringBuilder builder = new StringBuilder();
builder.append("http://bocichina.com/bci_path?id=0");
action.setData(Uri.parse(builder.toString()));
startActivity(action);

  • 判断app是否安装

    //判断是否安装了应用
    private boolean schemeValid() {

        PackageManager manager = getApplicationContext().getPackageManager();
        Intent action = new Intent(Intent.ACTION_VIEW);
        action.setData(Uri.parse("http://bocichina.com/bci_path"));
        List list = manager.queryIntentActivities(action, PackageManager.GET_RESOLVED_FILTER);
        return list != null && list.size() > 0;
}

  • 在 Capabilities -> Associated Domains 添加跳转域名格式为 applinks:universal-link.example.com

​ ![image-20220415210732163](/Users/tzt/Library/Application Support/typora-user-images/image-20220415210732163.png)

后端改造点

  • 添加对通用链接的支持很容易。您需要采取三个步骤：

    • 创建一个apple-app-site-association文件，其中包含有关您的应用可以处理的 URL 的 JSON 数据。
    • 将文件上传apple-app-site-association到您的 HTTPS Web 服务器。您可以将文件放在服务器的根目录或.well-known子目录中。
    • 准备您的应用以处理通用链接。

  • 配置文件如下

    {
  "applinks": {
      "details": [
           {
             "appIDs": [ "ABCDE12345.com.example.app", "ABCDE12345.com.example.app2" ],
             "components": [
               {
                  "#": "no_universal_links",
                  "exclude": true,
                  "comment": "Matches any URL whose fragment equals no_universal_links and instructs the system not to open it as a universal link"
               },
               {
                  "/": "/buy/*",
                  "comment": "Matches any URL whose path starts with /buy/"
               },
               {
                  "/": "/help/website/*",
                  "exclude": true,
                  "comment": "Matches any URL whose path starts with /help/website/ and instructs the system not to open it as a universal link"
               },
               {
                  "/": "/help/*",
                  "?": { "articleNumber": "????" },
                  "comment": "Matches any URL whose path starts with /help/ and which has a query item with name 'articleNumber' and a value of exactly 4 characters"
               }
             ]
           }
       ]
   },
   "webcredentials": {
      "apps": [ "ABCDE12345.com.example.app" ]
   },

    "appclips": {
        "apps": ["ABCED12345.com.example.MyApp.Clip"]
    }
}

  • apple-app-association的更新时机

    • 每次App安装后的第一次Launch，会拉取apple-app-association

    • Appstore每次App的版本更新后的第一次Launch，也会更新apple-app-association

    • 新版本下载方式变更 iOS14以及以后版本 链接

      在 macOS 11 及更高版本和 iOS 14 及更高版本中，应用程序apple-app-site-association从 Apple 管理的内容交付网络 (CDN) 专门针对关联域请求文件，而不是直接从您的 Web 服务器请求文件。如果 CDN 有该文件的旧版本，或者还没有该文件的副本，它会连接到您的 Web 服务器以获取最新版本。

    • 新版本更新时机 链接

      Apple 的内容交付网络会apple-app-site-association在 24 小时内为您的域请求文件。设备在应用安装后大约每周检查一次更新。

浏览器触发universal-link条件说明

  • 手机安装app/支持universal-link/并且打开过
  • 跳转的 url 是否和当前页面的 url 为不同域名，才能触发 universal-link机制

备注:app启动后,系统会检索是否开启 Associated Domains，如果开启去下载 apple-app-site-association 文件,然后对此 APP 进行标注。用户加载链接时系统会先对已标注的 APP 进行检索，如果匹配则回对应打开 APP

App-link(应用链接)

应用链接描述

Android App Links是一种特殊的Deep Links，它使Android系统能够直接通过网站地址打开应用程序对应的内容页面，而不需要用户选择使用哪个应用来处理网站地址。

要添加Android App Links到应用中，需要在应用里定义通过Http(s)地址打开应用的intent filter，并验证你确实拥有该应用和该网站。如果系统成功验证到你拥有该网站，那么系统会直接把URL对应的intent路由到你的应用。

前端改造点

  • 在AndroidManifest里要求系统自动进行App Links的所有权验证。这个配置会告诉Android系统去验证你的应用是否属于在intent filter内指定的URL域名

      <!--要想在别的App上能成功调起App，必须添加intent过滤器-->
            <intent-filter android:autoVerify="true">
                <data
                    android:host="ouhuangapp.github.io"
                    android:scheme="https"
                    android:pathPrefix="/github.io" />
                <!--下面这几行也必须得设置-->
                <category android:name="android.intent.category.DEFAULT" />
                <action android:name="android.intent.action.VIEW" />
                <category android:name="android.intent.category.BROWSABLE" />
            </intent-filter>
            <intent-filter>
                <action android:name="android.intent.action.VIEW" />

                <category android:name="android.intent.category.DEFAULT" />
                <category android:name="android.intent.category.BROWSABLE" />

                <data
                    android:scheme="https"
                    android:host="ouhuangapp.github.io"
                    android:path="/github.io" />
            </intent-filter>

微信开放标签

功能介绍

场景:用户在微信内访问网页时需要跳转到 APP,比如今日头条。

实现条件

绑定域名和移动应用后，即可在网页中使用“微信开放标签”跳转对应的移动应用

问题思考

H5引入js,如何知道是否安装了app？

结论：页面是不知道的,只能提示打开app，用户点击后，js可以可以使用微信用原生交互(url-scheme)，校验完成回掉给H5，最终决定是否打开app还是跳转appstore

方案总结

app唤醒场景

1.打开微信里面的小程序

2.打开头条里面的新闻详情

3.微信里面打开其他app

4.小程序里面打开其他app

5.浏览器里面打开其他app

以上场景均为唤醒app后做的业务

app打开目前流行的方案

  • iOS

    • 使用Univerlink方式
    • 在微信浏览器内使用微信标签(微信屏蔽了url-scheme,新版本sdk支持Univerlink)
    • 在系统浏览器使用URL-scheme

  • android

    • 使用URL-shcheme(deep link)
    • 在微信浏览器内使用微信标签(微信屏蔽了url-scheme) 需要引导用户到系统浏览器打开(最终是URL-scheme)
    • 在系统浏览器使用URL-scheme
    • app-link 兼容性有问题部分浏览器打不开(百度浏览器-oppo A32) 有些需要用户点击
